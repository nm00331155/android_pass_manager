# KeyPass (SecureVault) Autofill 修正指示書

## 概要

本アプリのAutofill機能において、**登録済みサイトで自動入力が正しく動作しない**致命的バグが存在する。
以下の問題を本指示書のとおりに修正すること。

`https://github.com/nm00331155/android_pass_manager` (branch: `main`)

## 修正1: RemoteViews のレイアウト不整合を修正（致命的）

### 問題

`buildFillResponse` メソッド内で `android.R.layout.simple_list_item_1` を使用している。
`RemoteViews(packageName, android.R.layout.simple_list_item_1)` は、第1引数が自アプリのパッケージ名であるのにリソースIDがAndroidフレームワーク側のものであるため、多くの端末で Dataset の inflate が失敗し、**候補をタップしてもフィールドに値が入力されない**。

修正2: 並行 onFillRequest の競合を防止（致命的）
問題
onFillRequest が短時間に複数回呼ばれた場合、全て serviceScope.launch で並行実行される。 Android Autofill フレームワークは新しい onFillRequest が発生すると前の FillCallback を無効化するが、 前のジョブが後から callback.onSuccess を呼ぶと無効な callback への応答となり、 新しいリクエストの応答が返されないケースが発生する。

修正内容
SecureVaultAutofillService クラスにジョブ管理用のプロパティを追加し、 新しいリクエストが来たら前のジョブをキャンセルする。

プロパティ追加
Copyclass SecureVaultAutofillService : AutofillService() {

    // ... 既存の @Inject フィールド ...

    private val serviceScope = CoroutineScope(SupervisorJob() + Dispatchers.IO)
    private var currentFillJob: Job? = null  // ← 追加
import kotlinx.coroutines.Job が必要になるので import に追加すること。

onFillRequest メソッド内の修正
Copyoverride fun onFillRequest(
    request: FillRequest,
    cancellationSignal: CancellationSignal,
    callback: FillCallback
) {
    logger.d(TAG, "=== onFillRequest START ===")

    // ★ 前のジョブをキャンセル
    currentFillJob?.cancel()

    val structure = request.fillContexts.lastOrNull()?.structure
    if (structure == null) {
        logger.w(TAG, "onFillRequest: structure is null")
        callback.onSuccess(null)
        return
    }

    val targetPackageName = structure.activityComponent?.packageName.orEmpty()
    logger.d(TAG, "onFillRequest: targetPackage=$targetPackageName")

    if (targetPackageName in EXCLUDED_PACKAGES || targetPackageName == packageName) {
        logger.d(TAG, "onFillRequest: skipping excluded package")
        callback.onSuccess(null)
        return
    }

    val targets = smartFieldDetector.detect(structure)
    logger.d(
        TAG,
        "onFillRequest: detected usernameId=${targets.usernameId}, passwordId=${targets.passwordId}, otpId=${targets.otpId}, webDomain=${targets.webDomain}"
    )

    if (targets.usernameId == null && targets.passwordId == null && targets.otpId == null) {
        logger.d(TAG, "onFillRequest: no fields detected")
        callback.onSuccess(null)
        return
    }

    val job = serviceScope.launch {
        // ... 既存の処理（変更なし） ...
    }

    // ★ 現在のジョブを保持
    currentFillJob = job

    cancellationSignal.setOnCancelListener {
        logger.d(TAG, "onFillRequest: cancelled")
        job.cancel()
    }
}

修正4: Dataset が無い場合の FillResponse 制御（重大）
問題
credentials が 0 件でも SaveInfo だけの FillResponse を返しているため、 全てのログインフォームで不要な保存ダイアログが表示される。

修正内容
buildFillResponse の最終 return 行を以下のように変更する。

現在のコード
Copyval result = if (hasDataset || saveInfo != null) responseBuilder.build() else null
修正後のコード
Copyval result = if (hasDataset) {
    saveInfo?.let { responseBuilder.setSaveInfo(it) }
    responseBuilder.build()
} else if (saveInfo != null) {
    // Dataset が無い場合でも SaveInfo を返す（未登録サイトの新規保存用）
    // ただし FillResponse.Builder に SaveInfo のみセットする
    responseBuilder.build()
} else {
    null
}
注意: 上記のロジック変更に伴い、saveInfo?.let { responseBuilder.setSaveInfo(it) } が現在 responseBuilder の初期化直後に呼ばれているが、この行を削除し、上記の分岐内でのみ setSaveInfo を呼ぶように変更すること。

具体的には buildFillResponse メソッド全体を以下の構造にする:

Copyprivate fun buildFillResponse(
    targets: SmartFieldDetector.DetectedFields,
    credentials: List<Credential>
): FillResponse? {
    val saveInfo = createSaveInfo(targets.usernameId, targets.passwordId)
    if (credentials.isEmpty() && saveInfo == null && targets.otpId == null) {
        logger.d(TAG, "buildFillResponse: no dataset and no saveInfo")
        return null
    }

    var hasDataset = false
    val responseBuilder = FillResponse.Builder()
    // ★ ここでは setSaveInfo を呼ばない（後で条件付きでセットする）

    credentials.forEach { credential ->
        try {
            val presentation = createPresentation(credential, locked = false)
            val datasetBuilder = Dataset.Builder(presentation)
            var hasValue = false

            targets.usernameId?.let { id ->
                datasetBuilder.setValue(id, AutofillValue.forText(credential.username))
                hasValue = true
                logger.d(TAG, "buildFillResponse: set username=${credential.username} for id=$id")
            }
            targets.passwordId?.let { id ->
                datasetBuilder.setValue(id, AutofillValue.forText(credential.password))
                hasValue = true
                logger.d(TAG, "buildFillResponse: set password for id=$id")
            }

            if (hasValue) {
                responseBuilder.addDataset(datasetBuilder.build())
                hasDataset = true
                logger.d(TAG, "buildFillResponse: added dataset id=${credential.id}")
            }
        } catch (e: Exception) {
            logger.e(TAG, "buildFillResponse: EXCEPTION building dataset", e)
        }
    }

    targets.otpId?.let { otpId ->
        try {
            val otpPresentation = createOtpPresentation()
            val otpIntent = createOtpAuthenticationPendingIntent(otpId)
            val otpDatasetBuilder = Dataset.Builder(otpPresentation)
            otpDatasetBuilder.setAuthentication(otpIntent.intentSender)
            otpDatasetBuilder.setValue(otpId, AutofillValue.forText(""))
            responseBuilder.addDataset(otpDatasetBuilder.build())
            hasDataset = true
            logger.d(TAG, "buildFillResponse: added otp dataset with auth")
        } catch (e: Exception) {
            logger.e(TAG, "buildFillResponse: EXCEPTION building OTP dataset", e)
        }
    }

    // ★ Dataset がある場合のみ SaveInfo もセットする
    if (hasDataset) {
        saveInfo?.let { responseBuilder.setSaveInfo(it) }
    } else if (saveInfo != null) {
        // Dataset は無いが SaveInfo のみ返す（未登録サイトの新規保存用）
        responseBuilder.setSaveInfo(saveInfo)
    }

    val result = if (hasDataset || saveInfo != null) responseBuilder.build() else null
    logger.d(
        TAG,
        "buildFillResponse: final result=${if (result != null) "FillResponse" else "null"}, hasDataset=$hasDataset, hasSaveInfo=${saveInfo != null}"
    )
    return result
}

制約事項
変更するファイルは SecureVaultAutofillService.kt のみ。 SmartFieldDetector.kt は今回変更不要。
CredentialDao.kt, CredentialRepositoryImpl.kt, CredentialEntity.kt, Credential.kt, レイアウト XML 等は一切変更しない。
新規ファイルの追加は不要。
import 文は必要に応じて追加・整理すること（kotlinx.coroutines.Job の追加が必要）。
既存のメソッドシグネチャ（createPresentation, createOtpPresentation, createAuthenticationPendingIntent, createOtpAuthenticationPendingIntent）は変更せずそのまま使用する。
コメントは日本語で記述する。
companion object 内の既存定数は値を変更しない。EXCLUDED_PACKAGES の追加のみ行う。
修正後の確認観点
Brave Browser で id.kuronekoyamato.co.jp にアクセスし、ログインフォームに KeyPass の候補が表示され、タップすると username/password が正しく入力されること
他のブラウザ（Chrome, Firefox 等）や一般アプリでも同様に自動入力が動作すること
Bitwarden 等のパスワードマネージャーアプリで候補が表示されない・保存ダイアログが出ないこと
未登録サイトでログイン後、保存ダイアログが表示され新規保存が動作すること
logcat で === onFillRequest END (success) === が出力され、エラーがないこと
