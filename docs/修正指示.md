## 問題 1: 「いいえ」ボタン文字色が白いまま

ユーザー添付スクリーンショットにより、**認証付き autofill ダイアログの「いいえ」ボタンが白背景に白文字で表示され、読めない**ことが確認されています。  
これは繰り返し指摘されており、優先度の高いUI不具合です。

### 期待する状態
- 「いいえ」が白背景上でも明確に読める
- light / dark / Samsung 端末差分でも再発しにくい
- 一部ダイアログだけでなく、**共通的に再発防止できる修正**になっている

### 対象候補
- `ConfirmDialog.kt`
- 認証付き autofill ダイアログを描画している composable
- 共通ボタンスタイル
- `ButtonDefaults`
- `TextButton` / `OutlinedButton`
- `MaterialTheme.colorScheme`
- dynamic color との組み合わせ

### やるべきこと
1. 画像の赤枠ダイアログを構成している composable / button 実装箇所を特定する
2. 「いいえ」ボタン文字色が theme 任せで白になっていないか確認する
3. 必要なら **明示的に可読な色** を指定する
4. 可能なら同様の確認ダイアログ全体に横展開する
5. 端末依存で崩れないかを確認する

### 禁止事項
- その場しのぎで1箇所だけハードコードして終わらないこと
- 無関係な theme 全面改修をしないこと

## 問題 2: パスキー登録は開始するが、エラーで完了しない

現状、Amazon の passkey 作成画面で SecureVault / KeyPass 側の作成UIまでは起動しています。  
つまり、問題は「provider が出ない」段階ではなく、**create フローの途中または応答生成で失敗している**段階です。

### 現状整理
- provider / chooser は出ている
- 作成ダイアログも出ている
- しかし、最終的にサイト側でエラーとなり、登録完了していない

### まず疑うべき箇所
- `CredentialProviderAuthActivity.kt`
- `PasskeyWebAuthnHelper.kt`
- passkey create request の parse
- RP ID / origin 補完
- `clientDataHash`
- WebAuthn response 生成
- `CreateCredentialResponse` 返却処理
- Activity result / exception handling
- Amazon 側との互換性
- Samsung / Android 14+ / Android 15+ 差分

### 対象候補
- `KeyPassCredentialProviderService.kt`
- `CredentialProviderAuthActivity.kt`
- `PasskeyWebAuthnHelper.kt`
- `provider.xml`
- Manifest の provider service 宣言
- 関連 logging 箇所

### やるべきこと
1. create フローで失敗する箇所をログベースで特定する
2. `CreateCredentialException` / `CreateCredentialUnknownException` / request parse failure を追う
3. `rpId`, `origin`, `clientDataHash`, challenge, response JSON の生成に不整合がないか確認する
4. Amazon 実行時のケースを基準に、最小差分で修正する
5. 「何が原因で失敗したか」を説明できるログを残す

### 期待する状態
- 少なくとも passkey create の失敗原因が明確になる
- 可能なら passkey 登録が最後まで完走する
- まだ無理な場合でも、OS制約なのか実装不備なのか切り分けできる
