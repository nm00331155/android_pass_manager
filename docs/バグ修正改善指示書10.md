SecureVault Autofill 修正指示書
概要
Autofill サービスおよび Credential Provider サービスが正常に候補を表示できない問題を修正する。ログ分析の結果、以下の3つの根本原因が特定された。

問題1: Credential Provider が空ユーザー名でクラッシュして全滅する
原因
KeyPassCredentialProviderService.onBeginGetCredentialRequest 内で、credentials.take(MAX_ENTRIES).forEach のループ中に username が空のクレデンシャルに遭遇すると PasswordCredentialEntry.Builder が IllegalArgumentException: username must not be empty を投げる。この例外が外側の catch に捕まり callback.onError(GetCredentialUnknownException()) が返され、成功済みの1件目も含めて全候補が破棄される。

該当ログ
18:56:23.941 D/KeyPassCredProvider: added entry id=1, user=502088, service=500445440a600d0794538046.smarthr.jp
18:56:23.941 E/KeyPassCredProvider: error
  java.lang.IllegalArgumentException: username must not be empty
修正対象ファイル
app/src/main/java/com/securevault/app/service/credential/KeyPassCredentialProviderService.kt

修正内容
onBeginGetCredentialRequest メソッド内の credentials.take(MAX_ENTRIES).forEach ループを以下のように変更する。

credential.username.isBlank() のチェックを追加し、空ユーザー名のクレデンシャルは return@forEach でスキップする
各エントリの PasswordCredentialEntry.Builder(...).build() と credentialEntries.add(...) を個別の try-catch で囲み、1件の失敗が全体を止めないようにする
修正後のforEachブロック:

Copycredentials.take(MAX_ENTRIES).forEach { credential ->
    if (credential.username.isBlank()) {
        logger.w(
            TAG,
            "onBeginGetCredentialRequest: SKIP credential id=${credential.id} service=${credential.serviceName} (empty username)"
        )
        return@forEach
    }

    try {
        credentialEntries.add(
            PasswordCredentialEntry.Builder(
                applicationContext,
                credential.username,
                createGetPendingIntent(credential.id),
                option
            )
                .setDisplayName(credential.serviceName)
                .build()
        )
        logger.d(
            TAG,
            "onBeginGetCredentialRequest: added entry id=${credential.id}, user=${credential.username}, service=${credential.serviceName}"
        )
    } catch (e: Exception) {
        logger.w(
            TAG,
            "onBeginGetCredentialRequest: failed to build entry id=${credential.id}",
            e
        )
    }
}

問題2: Autofill Service の非同期コールバックがキャンセルされている
原因
onFillRequest 内で serviceScope.launch { ... } によりコルーチンで非同期処理を行い、withContext(Dispatchers.Main) で callback.onSuccess(response) を呼んでいる。Brave ブラウザが短時間に3回連続で onFillRequest を発行するため、前のリクエストの CancellationSignal が発火し、コルーチンの job.cancel() が呼ばれる。結果として withContext(Dispatchers.Main) 内の callback.onSuccess が CancellationException で到達不能になる。

DB アクセスは数ミリ秒で完了するため、Autofill の5秒タイムアウト内に十分収まる。runBlocking で同期的に処理してOSに確実にレスポンスを返す方式に変更する。

該当ログ
18:55:40.245〜18:55:40.284 に3つの onFillRequest が同時並行で処理されている
修正対象ファイル
app/src/main/java/com/securevault/app/service/autofill/SecureVaultAutofillService.kt

修正内容
onFillRequest メソッド全体を以下に置き換える。serviceScope.launch と CancellationSignal のハンドリングを削除し、runBlocking(Dispatchers.IO) で同期的にDB検索を行い、直接 callback.onSuccess(response) を呼ぶ。デバッグ用の Toast も削除する。

Copyoverride fun onFillRequest(
    request: FillRequest,
    cancellationSignal: CancellationSignal,
    callback: FillCallback
) {
    logger.d(TAG, "=== onFillRequest START ===")

    val structure = request.fillContexts.lastOrNull()?.structure
    if (structure == null) {
        logger.w(TAG, "onFillRequest: structure is null")
        callback.onSuccess(null)
        return
    }

    val targetPackageName = structure.activityComponent?.packageName.orEmpty()
    logger.d(TAG, "onFillRequest: targetPackage=$targetPackageName")

    if (targetPackageName in EXCLUDED_PACKAGES || targetPackageName == packageName) {
        logger.d(TAG, "onFillRequest: skipping excluded package=$targetPackageName")
        callback.onSuccess(null)
        return
    }

    val targets = smartFieldDetector.detect(structure)
    showBrowserSettingsGuideIfNeeded(targetPackageName)
    logger.d(
        TAG,
        "onFillRequest: detected usernameId=${targets.usernameId}, passwordId=${targets.passwordId}, otpId=${targets.otpId}, webDomain=${targets.webDomain}"
    )

    if (targets.usernameId == null && targets.passwordId == null && targets.otpId == null) {
        logger.d(TAG, "onFillRequest: no fields detected")
        callback.onSuccess(null)
        return
    }

    val inlineRequest = if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
        request.inlineSuggestionsRequest
    } else {
        null
    }

    try {
        if (targets.otpId != null) {
            runCatching {
                smsOtpManager.startListening()
            }.onFailure { throwable ->
                logger.w(TAG, "Failed to start SMS OTP listener", throwable)
            }
        }

        val credentials = if (targets.usernameId != null || targets.passwordId != null) {
            runBlocking(Dispatchers.IO) {
                runCatching {
                    resolveCredentials(targetPackageName, targets.webDomain)
                }.getOrElse { throwable ->
                    logger.e(TAG, "Failed to resolve autofill credentials", throwable)
                    emptyList()
                }
            }
        } else {
            emptyList()
        }

        logger.d(TAG, "onFillRequest: resolved ${credentials.size} credentials")
        credentials.forEachIndexed { index, credential ->
            logger.d(
                TAG,
                "onFillRequest: credential[$index] id=${credential.id}, service=${credential.serviceName}, user=${credential.username}"
            )
        }

        val response = buildFillResponse(targets, credentials, inlineRequest)
        logger.d(TAG, "onFillRequest: responseBuilt=${response != null}")

        callback.onSuccess(response)
        logger.d(TAG, "=== onFillRequest END (success) ===")
    } catch (throwable: Throwable) {
        logger.e(TAG, "onFillRequest: exception", throwable)
        callback.onSuccess(null)
    }
}

問題3: buildFillResponse の RemoteViews が独自レイアウトを使っていない
原因
buildFillResponse 内で RemoteViews(packageName, android.R.layout.simple_list_item_1) を使用しているが、アプリには autofill_suggestion_item.xml という専用レイアウトが既に存在する。android.R.layout.simple_list_item_1 はシステムリソースであり、一部のOSバージョンやメーカーカスタムUIで正しく表示されない場合がある。

修正対象ファイル
app/src/main/java/com/securevault/app/service/autofill/SecureVaultAutofillService.kt

修正内容
buildFillResponse メソッド内の credentials.forEachIndexed ループにある usernamePresentation と passwordPresentation の RemoteViews 構築を以下に変更する。

変更前:

Copyval usernamePresentation = RemoteViews(packageName, android.R.layout.simple_list_item_1).apply {
    setTextViewText(
        android.R.id.text1,
        "${credential.serviceName} - ${credential.username}"
    )
}
val passwordPresentation = RemoteViews(packageName, android.R.layout.simple_list_item_1).apply {
    setTextViewText(android.R.id.text1, "パスワード (${credential.serviceName})")
}
変更後:

Copyval usernamePresentation = RemoteViews(packageName, R.layout.autofill_suggestion_item).apply {
    setTextViewText(R.id.service_name, credential.serviceName)
    setTextViewText(R.id.username, credential.username)
}
val passwordPresentation = RemoteViews(packageName, R.layout.autofill_suggestion_item).apply {
    setTextViewText(R.id.service_name, credential.serviceName)

}
