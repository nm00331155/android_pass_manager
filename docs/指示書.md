## 最重要方針

- **既存実装を壊さないことを最優先**にしてください。
- 実装前に必ず、**コード・docs・manifest・navigation・repository・autofill・credential provider 周辺を読む**こと。
- README は古い可能性が高いため、**`docs/specification.md` と `docs/status.md` を優先**して現状把握してください。
- 推測で大規模変更しないこと。
- **最小差分**で考えること。
- **INTERNET 権限は追加しないこと。**
- 暗号・保存・認証仕様は、明示的な必要性と影響確認なしに変更しないこと。
- Autofill / Credential Provider / passkey / OTP は、Androidバージョン差・ブラウザ差・端末差の影響が大きいため、軽率に共通化や単純化をしないこと。
- 変更が必要な場合は、必ず**理由・影響範囲・検証観点**を示すこと。
- 必要なら **`docs/specification.md`** と **`docs/status.md`** を更新してください。
